グレプタイルのボットが従う「カスタムコンテキスト」を指定することで、動作をカスタマイズできます。コンテキストには、カスタムルール、スタイルガイドやドキュメント、グレプタイルにとって有用な各種情報を含められます。

各ルールやガイドなどは、特定のリポジトリや特定のファイルパス/パターンにのみ適用されるようにスコープを設定できます。 


## カスタムコンテキストの種類

カスタムコンテキストには3種類あります。

### カスタムルール
「リスト内包表記ではなくループを使う」などのルールを定義でき、グレプタイルがPR全体でこのルールを適用します。

例:
    ```
    GraphQLコードで一貫性があり、意味のある命名規則を用いる
    ```
    ```
    Goの慣用的なパターンや規約に従う
    ```
    ```
    エラーを一貫して明示的に処理する
    ```

### スタイルガイドとドキュメント
既存のスタイルガイドやドキュメントをグレプタイルに指定でき、PRレビュー時のコンテキストとして利用します。

```
src/docs/styleguide.md
```

### その他のコンテキスト
チームやコードベースの特有事項を記述して、グレプタイルをパーソナライズするための自由記述欄もあります。

例:
    ```
    課金ロジックが次のルールに従うことを確認する:
    - ユーザーアカウントは90日間の非アクティブ後に無効化する
    - $1000を超える注文にはマネージャーの承認が必要
    - 返金は48時間以内に処理すること
    - プレミアムユーザーは優先キューにアクセスできるようにする
    ```


---

## スコープ

各種のコンテキスト、ルール、ガイド、および「その他」は、特定のリポジトリやファイルパターンにのみ適用されるようにスコープを設定できます。

たとえば、`prop types` の使用を強制したいが、それを `acme/web_app` に限定し、さらにパターン `src/frontend/*.tsx` に一致するファイルにのみ適用したい場合があります。

<Note>
スコープを明示的に定義しない場合、グレプタイルがルールやスタイルガイドなどを適用するタイミングを自動的に判断します。
</Note>

## 学習

グレプタイルは、チーム内の他の開発者によるPRへのコメントや、グレプタイルのコメントへの返信、👍/👎のリアクションといったインタラクションデータを基に、ルールを学習・推定できます。

グレプタイルの最新の学習内容は、ダッシュボードのカスタムコンテキストページで確認できます。
