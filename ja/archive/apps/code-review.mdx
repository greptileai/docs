### なぜ有用か？

Greptile のコードレビューボットは次のようなコメントを書きます:

- 変更点をコードベースの文脈とともに要約し、人間のレビュアーが変更を理解・把握しやすくします。
- 文脈を踏まえて、変更が広範なコードベースに与える影響を議論できます。
- 変更によって生じうるパフォーマンス、セキュリティ、コンプライアンス上の問題を特定できます。
- 変更に関連する既存の類似コードや抽象化を参照し、重複を防ぎます。
- あなたのカスタムスタイルガイドや規定のパターンを順守します。

つまり、
1. PR のレビューやレビュー待ちに費やす時間が減り、コードを書く時間が増えます。
2. 人間が見落としがちなバグをより多く検出できます。
3. 人間のレビュアーは変更の「全体像」や「戦略的」側面に集中でき、ボットは「戦術的」側面に集中できます。

### インストール

PR レビューボットのインストールは[こちら](https://app.greptile.com/login)をクリックしてください。

### トリガー

デフォルトでは、Greptile は新しい PR が最初にオープンされたとき、またはユーザーが PR に `@greptileai` とコメントしたときにレビューを実行します。特定のトリガーで PR をレビューするように設定することもできます。
- 特定のラベルが付いた新規 PR のみをトリガー。
- PR ブランチにプッシュされた新規コミットごとにトリガー。
- 特定のラベルが付いている、またはタイトルや説明に特定のキーワードを含む PR はトリガーしない。

### `greptile.json` を用いたリポジトリ単位の設定とパターンリポジトリ（patternRepositories）

デフォルトブランチのリポジトリルートに `greptile.json` ファイルを追加すると、各リポジトリごとにカスタム設定を適用できます。
Settings パネル右上の `copy` または `download` アイコンをクリックすると、[こちら](https://app.greptile.com/login)から正しい形式のグローバル設定を取得できます。
注: `ignorePatterns` フィールドは `.gitignore` の[構文](https://git-scm.com/docs/gitignore)に準拠します。

```json greptile.json
{
  "labels": [
    "feature",
    "bug"
  ],
  "comment": "Disclaimer: This is AI-generated.",
  "commentTypes": [
    "logic",
    "syntax"
  ],
  "instructions": "Ensure style_guide.md is enforced.",
  "ignoreKeywords": "rename\nlinter\nprettier\ngreptile-ignor",
  "ignorePatterns": "testing/**/*.py\n*.md\n*.txt\n*.json",
  "patternRepositories": ["acme/backend"],
  "triggerOnUpdates": true,
  "shouldUpdateDescription": false,
  "disabledLabels": [
    "docs"
  ],
  "includeAuthors": [
    "dakshgup",
    "schoi"
  ],
  "excludeAuthors": [
    "cool-dev"
  ]
}
```

**パターンリポジトリ（Patterns repos）**

`greptile.json` では、レビュー対象に関連するリポジトリを `patternsRepo` フィールドで指定し、レビューに有用なコンテキストを追加できます。
例として、バックエンドリポジトリの `greptile.json` にフロントエンドリポジトリを指定しておくと、ボットはバックエンドコードをレビューする際にフロントエンドのコードも参照できます。

詳細は [greptile.com/code-review-bot](https://www.greptile.com/code-review-bot) を参照してください。
