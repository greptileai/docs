### Greptile は何にコメントしますか？

Greptile は多様なバグ、アンチパターン、スタイル違反に標準で対応し、カスタムルールの適用も構成できます。 

Greptile はスタックに依存しない AI 駆動のリンターと考えられます。

<Card title="Logic" icon="lightbulb" horizontal> 
    変更の意図を推測し、関連ファイルやインポートなどを走査して論理的な誤りを可視化します。
</Card>

<Card title="Style" icon="code" horizontal> 
    スタック固有のベストプラクティスに加え、あなたのカスタムスタイルガイドやパターンを遵守させます。
</Card>

<Card title="Performance" icon="gauge-high" horizontal>
    変更がコードベースに与える影響を分析し、潜在的なパフォーマンス問題を特定します。
</Card>

<Card title="Security" icon="shield" horizontal>
    変更がコードベースに与える影響を分析し、潜在的なセキュリティ問題を特定します。
</Card>

<Card title="Compliance" icon="list-check" horizontal>
    カスタムのコンプライアンスポリシーや規制の遵守を強制します。
</Card>

<Card title="Code Repetition" icon="copy" horizontal>
    コードベースを走査して類似コードを見つけ、潜在的な重複を特定します。
</Card>


### なぜ有用なのですか？

Greptile のコードレビュー ボットは次のようなコメントを書きます:

- 文脈を理解し、変更が広範なコードベースに与える影響を議論できます。
- 変更によって生じうるパフォーマンス、セキュリティ、コンプライアンス上の問題を特定できます。
- 重複を防ぐため、変更に関連する既存の類似コードや抽象化を参照します。
- あなたのカスタムスタイルガイドと規定パターンの遵守を徹底します。

つまり:
1. PR のレビューやレビュー待ちに費やす時間が減り、コードを書く時間が増えます。
2. 人間が見落としがちなバグをより多く検出します。
3. 人間のレビュアーは変更の「全体像」や「戦略的」側面に集中でき、ボットは「戦術的」側面に集中できます。
