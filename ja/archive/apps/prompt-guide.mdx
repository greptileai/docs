## 導入

多くの LLM 搭載ツールと同様に、Greptile で具体的に何ができるのか把握するのは難しいことがあります。このページでは、開発者に好評なユースケースをいくつか紹介します。本ガイドは主に当社の[コードベース Q&A ウェブアプリ](https://app.greptile.com/login)を想定していますが、API を含むその他のインターフェースにも同様に適用できます。 


### 複雑なコードベースの理解

Greptile は、リポジトリ全体の仕組みや特定部分の動作を、わかりやすい説明と具体的なコード参照リンクによって理解する手助けをします。 

*例* 

1. 「このコードベースでは認証はどのように機能していますか？」
2. 「API 経由で送信されるユーザー入力をクレンジングしているコードはどこにありますか？」
3. 「`api/proc.ts` ファイルは何をしますか？」

<Tip>多くの開発者が、業務に関連するオープンソースのリポジトリで Greptile を使い、そこでの実装方法を学んでいます。</Tip>

### デバッグ

エラーメッセージ、スタックトレース、あるいは予期しない挙動の説明があれば、Greptile は原因を特定し、修正済みのコード例とともに解決策を提案できます。 

*例*
1. 「`signal.parse()` を呼び出すと `FileNotFound: Object has no attribute c://` エラーが出るのはなぜですか？」
2. 「プロフィールのドロップダウンをクリックするとページがリフレッシュされます。なぜですか？」
3. 「`[...Stack trace/error messages]` — なぜ起きていますか？」

### 複雑なコード変更の洗い出し

タスクやチケットの説明に基づき、Greptile はコードベースの理解を用いて、タスク達成に必要な手順を整理できます。必要なコードを生成し、コードベースのどこに配置すべきかも具体的に示します。 

*例*
1. 「サイドバーのトグル状態をローカルストレージに保存するにはどうすればよいですか？」
2. 「ユーザーが過去のチャット名を変更できるように、チャット履歴をどう修正すればよいですか？」
3. 「session token に対応できるよう、express のルーターの修正を手伝ってもらえますか？」

### サードパーティライブラリとの統合

同じ Greptile セッションに複数のリポジトリを追加できます。たとえば、同じセッションに `your-org/your-backend` と `nextauthjs/next-auth` を追加して、「`nextauth` を使って `your-backend` に Google サインインを追加するにはどうすればよいですか？」と尋ねられます。

*例*
1. `stripe/stripe-node` を追加して、「`your-backend` に従量課金を追加するにはどうすればよいですか？」と尋ねる
2. `twilio/twilio-ruby` を追加して、「`your-frontend` から twilio SDK へ LLM の出力をストリーミングするにはどうすればよいですか？」と尋ねる

<Tip>アプリのバグがオープンソースの依存関係に起因しているかもしれないと疑っていますか？そのリポジトリをセッションに追加して、Greptile が診断に役立つよう検索できるようにしましょう。</Tip>

### パフォーマンスとセキュリティの評価

Greptile は、パフォーマンスやセキュリティの改善点を見つける手助けができます。ファイルやディレクトリを指定して質問するだけです。

*例*

1. 「`api/auth` にセキュリティ上の懸念はありますか？」
2. 「パーサをより高速にするにはどうすればよいですか？」
3. 「`db_connector` をよりスケーラブルにするにはどう変更すればよいですか？」

<Warning>Greptile は LLM を基盤としているため、セキュリティ上の懸念に関しては網羅的に回答できない場合があります。回答は絶対的な正解ではなく、指針として扱ってください。 </Warning>

## 目的を持ったプロンプト設計

一般的に、Greptile は指示が具体的で、可能な限り多くの情報が提供されているほど高い性能を発揮します。

**ヒント:**

<Check>**シンボルを明記する:** 回答がコードベースのどこにあるか見当がつく場合は、シンボル名を含めることで Greptile がより適切に回答を見つけやすくなります。</Check>
<Check>**詳細を含める:** 可能な限り詳しく書いてください。デバッグ時は、エラーメッセージやスタックトレースなど、コンソールから得られる情報をすべて提供してください。</Check>
<Check>**出力形式を具体的に示す:** コードの生成が必要な場合は「generate the full code for this」と明示してください。短い回答がよければ「Answer in one sentence.」と指定してください。</Check>

他の LLM へのプロンプト作成と同様に、時間の経過とともに、あなたのコードベースやユースケースで Greptile を最適に活用するための細かなコツが見つかるはずです。プロンプトに関する癖を報告したい場合は、[ぜひお知らせください](mailto:support@greptile.com)。Greptile の改善に役立ちます。
