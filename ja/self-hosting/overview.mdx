このセクションでは、[akupara リポジトリ](https://github.com/greptileai/akupara)を使用して Greptile をセルフホストするための包括的なガイドを提供します。デプロイ方法は主に `docker-compose` と `kubernetes` の2通りをご用意しています。より簡便で柔軟なため、`docker-compose` の使用を強く推奨します。

<Info>
  akupara リポジトリには、Greptile をオンプレミスで実行するためのすべてのインフラコードとデプロイガイドが含まれています。必ず [GitHub](https://github.com/greptileai/akupara/releases) から最新のリリースをダウンロードしてください。
</Info>

## 前提条件

### 共通要件

- [akupara リポジトリ](https://github.com/greptileai/akupara) へのアクセス
- Linux ベースのデプロイ環境
- Docker イメージ取得のためのネットワークアクセス
- HTTPS エンドポイント用の SSL 証明書

### Docker Compose 要件

- Docker および Docker Compose がインストール済み
- 最低 8GB の RAM、CPU コア 4 基
- 100GB 以上の空きディスク容量

### Kubernetes 要件

- `terraform`、`helm`、`kubectl` がインストール済み
- AWS へのデプロイの場合：適切な認証情報を設定した `awscli`
- Kubernetes クラスター（EKS、GKE、AKS、または自前管理）へのアクセス
